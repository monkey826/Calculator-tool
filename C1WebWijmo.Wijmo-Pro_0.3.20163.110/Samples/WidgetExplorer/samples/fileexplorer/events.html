Events


Wijmo FileExplorer widget provides following events:

  • errorOccurred - fired when an error occurs.
  • fileOpened - fired after a file is opened.
  • fileOpening - fired before a file is opened.
  • filtered - fired after filtering.
  • filtering - fired before filtering.
  • folderChanged - fired after current active folder is changed.
  • folderLoaded - fired after a folder content is loaded.
  • itemCopied - fired after an item is copied.
  • itemCopying - fired before an item is copied.
  • itemDeleted - fired after an item is deleted.
  • itemDeleting - fired before an item is deleted.
  • itemMoved - fired after an item is moved.
  • itemMoving - fired before an item is moved.
  • itemPasted - fired after an item is pasted.
  • itemPasting - fired before an item is pasted.
  • itemRenamed - fired after an item is renamed.
  • itemRenaming - fired after an item is renamed.
  • itemSelected - fired after the node is selected.
  • newFolderCreated - fired after a new folder is created.
  • newFolderCreating - fired before a new folder is created.

In this demo, when performing different actions, the log will list the wijfileexplorer events and target folder/file.
