Introduction

When we created Wijmo 5, we decided it was time to take full advantage of the latest technologies and launch a pure HTML5/JavaScript product, no longer making compromises in order to support legacy browsers.

We also decided to leverage our experience in control development and implement some .NET tried-and-true concepts including the ICollectionView interface, event patterns, and control templates.

This application highlights some of the main features in Wijmo 5. If you want to learn more about the details, feel free to check the on-line documentation.

What's in Wijmo 5?

  • Core: Globalization, data binding, OData, tooltips, Control base class, Event class, and general utilities.
  • Input: AutoComplete, ComboBox, InputNumber, InputDate, InputTime, InputMask, InputColor, ListBox, Menu, MultiSelect, Calendar, ColorPicker, and more.
  • Grid: A brand new version of the FlexGrid control, more flexible and powerful than ever.
  • Chart: A brand new FlexChart control that is beautiful, powerful, and easy to use.
  • Gauge: A set of Gauge controls designed to display data effectively on desktop and on mobile devices.
  • Extensions: Many Wijmo controls can be extended with additional modules that add specialized functionality to the base controls. For example, the FlexChart has extensions for analytics, animations, and interaction; the FlexGrid has extensions for filtering, rows details, and grouping UI.

What are the requirements?

  • Modern Browsers (IE9+, ECMAScript 5)
  • Wijmo 5 has zero dependencies

Why require ECMAScript5?

  • Real properties (with getters/setters)
  • JSON support
  • SVG graphics
  • Better arrays, bind method, etc.
  • Smaller and more efficient (no hacks to support legacy browsers)
